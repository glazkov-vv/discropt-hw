# Как запускать

Данные о тестах и порогах указываются в файле `tests.txt`

Для запуска тестов используется скрипт `build_and_run.sh`. Результат выводится на экран и сохраняется в файле `report.txt`

# Отчет о работе

## Попытка 1. Жадный алгоритм. 

Несколько запусков жадника, который случайно может взять не самую оптимальную вершину.

Версия, которая с заданными вероятноятями берет один из трех ближайших соседей, берет 3 простых теста.

## Попытка 2. Муравьиный алгоритм

Стандартный муравьиный алгоритм. Для ускорения работы ограничиваем поиск k предпосчитанными глобально ближайшими точками (при отсутствия таковых непосещенных муравей берет глобально ближайшую из непосещенных)

Вероятность перехода по ребру пропорциальна произведению феромона на нем и обратному к длине ребра (в степенях $\alpha$ и $\beta$ соответственно).

Феромон на ребре "затухает" экспоненциально.

Исходно это решение пробивало 5 простых порогов.

После добавления локального спуска (перебираем пары ребер, меняем местами если можно поменять пары вершин в ребре) получилось пробить дополнительно 2 сложных порога.

Подкрутив константы (размер колонии муравьев на итерации = 50, Alpha=4, Beta=1, K=5), получилось взять третий сложный тест

